Try et catch

La version javascript 1.5 a introduit un gestionnaire d'exception avec les instructions try & catch. Le principe est simple. L'instruction try désigne le code qui doit être testé. Si aucune erreur n'est survenue le bloc de commandes de catch est simplement ignoré. Si par contre, une exception ou erreur est détectée, le contrôle de l'exécution passe directement au bloc de commande désigné par l'instruction catch. Le type d'exception est identifié dans une variable(indentificateur). A la fin du bloc catch, le programme continue son exécution par l'instruction qui suit. 

	try
	{
	//instruction à tester
		}
		catch(identificateur)
		{
	//instruction alternative
	}
